# مصروفاتي v2.3

نسخة محسنة من تطبيق مصروفاتي.

## الجديد

- وضع الخصوصية: يخفي الأرقام والمبالغ من الواجهة.
- منع تكرار عملية البنك لو وصلت كـ SMS وكإشعار في نفس الوقت.
- سجل العمليات بقى أصغر وأنضف، وكل عملية يظهر بجانبها العملة والتصنيف.
- الفلوس اللي ليك واللي عليك بقى فيها اختيار العملة: ريال سعودي أو جنيه مصري.
- كارت الدخل الإضافي بقى قابل للضغط لإضافة دخل منفصل للشهر.
- اختيار الفئة في تعديل العملية بقى من قائمة Scroll بدل الكتابة.
- أسفل التطبيق بقى فيه أزرار ثابتة: إضافة، السجل، الرئيسية، الفلوس، المزيد.

## مهم للتحديثات

تم إضافة مفتاح توقيع ثابت داخل المشروع: `app/masrofaty-release.jks`.
ده يخلي أي APK جديد يتثبت فوق القديم بدل ما يطلع خطأ اختلاف التوقيع.

## تفعيل التحديثات التلقائية

قبل البناء، افتح:

`app/src/main/res/values/strings.xml`

وحط رابط ملف `update.json` في:

```xml
<string name="update_json_url">https://raw.githubusercontent.com/USER/REPO/main/update.json</string>
```

المستخدم العادي هيشوف زر فحص التحديث فقط، والتطبيق بيفحص تلقائيًا عند الفتح.

## تفعيل Google Sync

قبل البناء، حط قيم Firebase في `strings.xml`:

```xml
<string name="firebase_api_key">...</string>
<string name="firebase_app_id">...</string>
<string name="firebase_project_id">...</string>
<string name="google_web_client_id">...</string>
```

بعدها المستخدم هيقدر يسجل دخول بحساب Google بزر واحد من داخل التطبيق.

## النسخ المحلي

التطبيق يحفظ نسخة تلقائية بعد التعديلات في:

- فولدر التطبيق على الموبايل
- ويحاول كمان يحفظ في Documents/Masrofaty/masrofaty_backup.json

لو التطبيق اتحدث، الداتا بتفضل موجودة. لو اتمسح وثُبت تاني، التطبيق يحاول يسترجع النسخة المحلية لو لقاها.

## v2.2 - إصلاح النسخ الاحتياطي بعد حذف التطبيق

مهم: أندرويد يحذف الداتا الداخلية لأي تطبيق عند عمل Uninstall. لذلك تم إضافة حلين عمليين:

1. Google Sync: استرجاع تلقائي بعد تسجيل الدخول بنفس حساب Google.
2. ملف نسخة احتياطية دائم: من داخل التطبيق افتح مزامنة Google وحفظ الداتا ثم اختر / أنشئ ملف حفظ دائم. التطبيق سيكتب عليه بعد كل تعديل. بعد حذف التطبيق وإعادة تثبيته استخدم استيراد نسخة من ملف واختر نفس الملف.

تمت إضافة Android Auto Backup وData Extraction Rules للمساعدة في استرجاع الداتا عبر نسخة الجهاز الاحتياطية، لكن الاعتماد الأقوى هو Google Sync أو ملف النسخة الاحتياطية الذي تختاره بنفسك.

## نظام التحديث التلقائي عبر GitHub - v2.4

النسخة دي فيها نظام تحديث مباشر للـ APK بدون Google Play:

1. ارفع المشروع على GitHub.
2. شغل GitHub Actions.
3. الـ Action هيعمل الآتي تلقائيًا:
   - يحط رابط `update.json` الصحيح داخل التطبيق حسب اسم الريبو والفرع.
   - يبني APK Release.
   - يعمل GitHub Release باسم رقم الإصدار، مثل `v2.4`.
   - يرفع ملف APK داخل الـ Release.
   - يحدث ملف `update.json` في الريبو برابط APK الجديد.
4. التطبيق عند المستخدمين يفحص التحديث تلقائيًا عند فتح التطبيق.
5. لو فيه إصدار جديد، يظهر للمستخدم زر `تحميل` فقط.

مهم: التثبيت خارج Google Play لا يمكن أن يتم بصمت. المستخدم لازم يوافق على تنزيل وتثبيت APK.

### كل تحديث جديد تعمل إيه؟

غير فقط في `app/build.gradle`:

```gradle
versionCode 16
versionName '2.5'
```

ثم اعمل Push على GitHub. الباقي يتم تلقائيًا من GitHub Actions.

### أين ملف update.json؟

الـ Action ينشئ أو يحدث الملف في جذر الريبو:

```text
update.json
```

والتطبيق يقرأه من رابط Raw GitHub تلقائيًا.
